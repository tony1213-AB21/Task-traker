# Daily Report MVP E2E 검증 보고 (2026-07-04)

라이브 Supabase 프로젝트를 대상으로 실제 세션·데이터로 수행한 종단 간 검증 결과입니다.
요구사항 기준 문서는 `goal/DAILY_REPORT_BUILD_GOAL.md`, 완료 기준은 그 문서의 Acceptance criteria입니다.

## 검증 환경

- 앱: 로컬 `npm run dev` (Next.js 15, App Router)
- DB/Auth: 원격 Supabase (`.env`의 `NEXT_PUBLIC_SUPABASE_URL`)
- 스키마: `supabase/schema.sql`을 사용자가 SQL Editor에서 실행, 9개 테이블 생성 확인
- 인증 키: `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` (service role 키 미사용)

## 스키마/RLS

- REST로 9개 테이블 조회 전부 `200`: profiles, projects, subtypes, tasks, entries, entry_tasks, entry_links, kpt_notes, user_preferences.
- 익명(세션 없음) 접근: `projects` select **0행**, insert **42501(RLS 위반) 거부**, `entries` select **0행**.

## 인증

- `signUp` 즉시 세션 발급 확인 (검증을 위해 Confirm email을 잠시 끈 상태에서 수행).
- 라우트 동작: `/login` 렌더(200), `/app` 미로그인 차단(307 → `/login`), `/auth/confirm` 잘못된 토큰 → 오류 리다이렉트(307 → `/login?error=confirm`).
- 앱 로그인 흐름은 `signInWithOtp`(매직링크) + `verifyOtp`. `verifyOtp` 경로는 `/auth/confirm` 라우트로 검증.

## 교차 사용자 RLS (User B가 User A 데이터 접근 시도)

- B가 A의 entry **읽기 0행**
- B가 A의 entry **수정 0행**
- B가 A의 project **삭제 0행**
- B의 시도 이후에도 A의 entry `content`='updated' 무결 유지

## 시간 로직 런타임 테스트 — 17/17 통과

실제 소스 `src/lib/time/format.ts`를 번들해 실행:

- Duration `HH:MM`: 35m→00:35, 60m→01:00, 80m→01:20, 815m→13:35
- 사람이 읽는 포맷: 815m→"13h 35m", 45m→"45m", 120m→"2h"

## 운영 메모 (Supabase 설정)

- **Confirm email**: E2E에서 rate limit을 우회해 즉시 세션을 얻기 위해 검증 중 잠시 껐고, 검증 완료 후 **다시 켬**. 프로덕션 로그인(매직링크/OTP)은 Confirm email이 켜진 상태를 전제로 정상 동작.
- **테스트 계정 삭제**: 검증에 사용한 `tony1213+e2e-a/b/c/d@abitus21.art` 계정을 Authentication → Users에서 **삭제 완료**. 관련 데이터는 이미 remaining=0.
